## Требования к PR

Необходимо в описании указать:

1. Какой функционал был добавлен в проект (с деталями, а не просто номер пункта)
2. Что было изменено в исходном проекте (если это требовалось для реализации нового функционала)
3. Как в приложении увидеть ваши нововведения (последовательность действий, список)
